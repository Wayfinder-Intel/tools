Privacy & Data Use

How these tools handle your data
← Back to Tools

All tools are static pages with no accounts and no analytics. Except where noted for map lookups/tiles, your inputs remain in your browser and are not transmitted.

Facebook Graph Builder (graph.html)

  • Data sent externally: None. Parsing and graph operations run locally in your browser.
  • Local storage: uses localStorage to persist Sessions/graph state (you can clear this any time).
  • Exports: JSON/CSV/PNG exports are generated in-browser for download.
  • Third‑party assets: D3 library via CDN. No tracking and no API calls.

Facebook Meta Mapper (map.html)

  • Data sent externally: place‑name text only is sent to OpenStreetMap’s Nominatim service for geocoding (converting names to coordinates).
  • Map tiles: basemap imagery loads from OpenStreetMap (OSM) tile servers.
  • Local storage: geocoding results are cached in localStorage to minimise repeat lookups.
  • Exports: JSON/CSV exports are generated locally from your filtered/mapped data.
  • Third‑party assets: Leaflet + MarkerCluster via CDN; OSM Tiles; OSM Nominatim.
  • Tip: avoid personal identifiers in the place text if you don’t want them sent to the geocoder.

Email Guesser (email.html)

  • Data sent externally: None. Address permutations are generated locally; copy/export happens in your browser.
  • Third‑party assets: None.

EXIF Metadata Inspector (metadata_mapper.html)

  • Local files: When you drag-and-drop or select a file from your computer, the file is read entirely in your browser. It is not uploaded or sent anywhere.
  • Image URLs (if used): If you enter an image URL, your browser will request that image from its host so it can be analyzed locally. In that case, the host serving the image will see a normal web request from your browser.
  • EXIF/metadata parsing: All parsing runs client-side via JavaScript. The tool does not call any cloud or third-party analysis API.
  • Timezone & display helpers: Conversions are computed in-browser using your device’s time zone.
  • Storage: The page may use localStorage for small UX preferences.
  • Third-party assets: The page may load static JS/CSS libraries from a CDN purely to serve the code.
  • Analytics & tracking: No analytics scripts are embedded. No tracking beacons are used.

Tip for OPSEC-sensitive use: Prefer local file uploads over URLs. If you must analyze by URL, consider saving the image locally first, then loading the file to keep all processing offline.

CDNs & Services Used

  • D3 (Graph Builder): served from a CDN.
  • Leaflet & MarkerCluster (Meta Mapper): served from a CDN.
  • OpenStreetMap Tiles (Meta Mapper): basemap imagery.
  • OpenStreetMap Nominatim (Meta Mapper): geocoding service for place‑name queries.

No analytics beacons or tracking scripts are embedded in these pages.
